# Скилл: SSH-настройка к серверу

Готовит SSH-доступ к серверу клиента для последующего деплоя.

## Что меняется в проекте
- `ssh/id_ed25519`, `ssh/id_ed25519.pub` — пара ключей (приватный в .gitignore)
- `ssh/config` — алиас проекта для удобного подключения
- `changelog.md` — запись о настройке

## Структура диалога

### 1. Спросить параметры сервера

> Подготовим SSH-доступ к серверу клиента. Введи:
> 1. IP-адрес или домен сервера (например, `203.0.113.42` или `bot.example.com`)
> 2. Пользователь на сервере (обычно `root` или `ubuntu` для VPS, `<имя>` если уже создан non-root)
> 3. Порт SSH (по умолчанию 22, на хардненом сервере часто меняют)

Запиши: `<host>`, `<user>`, `<port>`.

### 2. Проверить наличие пары ключей

```bash
ls <root>/ssh/id_ed25519 2>/dev/null
```

Если уже существует — спроси:
> Ключи `ssh/id_ed25519` уже есть. Использовать их или сгенерить заново? (лучше использовать, если на сервер уже залит)

Если решили заново — `rm <root>/ssh/id_ed25519*`.

### 3. Сгенерировать пару (если нет)

```bash
ssh-keygen -t ed25519 -f <root>/ssh/id_ed25519 -N "" -C "{{PROJECT_SLUG}}-deploy"
chmod 600 <root>/ssh/id_ed25519
chmod 644 <root>/ssh/id_ed25519.pub
```

### 4. Залить публичный ключ на сервер

Покажи клиенту публичный ключ и **попроси залить его на сервер** одним из способов:

```bash
cat <root>/ssh/id_ed25519.pub
```

**Способ A — через ssh-copy-id (если есть парольный доступ):**
```bash
ssh-copy-id -i <root>/ssh/id_ed25519.pub -p <port> <user>@<host>
```

**Способ B — вручную:**
1. Подключись к серверу любым способом (пароль, веб-консоль провайдера).
2. На сервере выполни:
   ```bash
   mkdir -p ~/.ssh && chmod 700 ~/.ssh
   echo "<содержимое id_ed25519.pub>" >> ~/.ssh/authorized_keys
   chmod 600 ~/.ssh/authorized_keys
   ```

Клиент должен подтвердить что ключ залит. Не двигайся дальше пока не залит.

### 5. Создать `ssh/config`

```bash
cat > <root>/ssh/config <<EOF
Host {{PROJECT_SLUG}}
    HostName <host>
    User <user>
    Port <port>
    IdentityFile <root>/ssh/id_ed25519
    IdentitiesOnly yes
    ServerAliveInterval 60
    ServerAliveCountMax 3
EOF
chmod 600 <root>/ssh/config
```

### 6. Тестовое подключение

```bash
ssh -F <root>/ssh/config {{PROJECT_SLUG}} 'hostname && whoami && uptime'
```

Если выдало hostname сервера — успех. Если нет — собери ошибку, разбирайся.

### 7. Записать в changelog.md

```
[YYYY-MM-DD HH:MM] SSH к серверу <host> настроен. Алиас: {{PROJECT_SLUG}}. Доступ: ssh -F ssh/config {{PROJECT_SLUG}}
```

### 8. Финальное сообщение

```
Готово. SSH-доступ к серверу <host> настроен.
- Подключение: ssh -F ssh/config {{PROJECT_SLUG}}
- Приватный ключ: ssh/id_ed25519 (НЕ КОММИТИТСЯ — в .gitignore)
- Публичный: ssh/id_ed25519.pub (можно коммитить)

Дальше — настрой безопасность сервера через скилл /server-hardening (UFW, fail2ban, отключение парольного логина).

Потом — задеплой через /deploy.
```

## Важно

- Никогда не коммитить приватный ключ. Проверь `.gitignore` (`ssh/*` исключение для `ssh/README.md`).
- Если у клиента уже есть SSH-доступ — спроси, использовать существующие ключи или сгенерить новые именно для этого проекта.
- Если сервер требует пароль и клиент не хочет ssh-copy-id — поддержи вариант ручной заливки.
