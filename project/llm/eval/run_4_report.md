## Что изменилось

Поле `expected_answer` больше не является инструкцией для judge. Теперь это полноценная модельная реплика, которую можно читать как эталонный ответ клиенту.

Такой формат ближе к обучающему примеру: в карточке прогона видно вопрос, эталон, фактический ответ бота и обоснование проверки.

## Наблюдения

- Новый формат эталонов лучше показывает качество ответа, а не только выполнение правил.
- Кейс неизвестной услуги LPG сначала выявил слабое место: бот предлагал связаться с администратором, но не давал контакт. Промпт усилен, после повторного прогона кейс прошел.
- Несколько кейсов проходят через LLM-judge, потому что ответ бота корректен по смыслу, но не совпадает с regex буквально.
